Monitor and Manage Linux Processes:

Defining a Process:

    A process is a running instance of a launched and executable program.

        Upon Creation, Consists of:

        1. An address of allocated memory

        2. Security properties including permissions and ownership

        3. One or more execution threads of program code

        4. A process state

   

        The environment of a process is a list of information including:

        1. Global and local variables

        2. Current scheduling context

        3. Allocated system resources (file descriptors, network ports)

   

Any process can fork in order to spawn a child process.

        fork := duplicate it's own address space

        child process := a process created by a parent process (RELATIVE)

Child Process Creation & Management:

1. Through a fork the CHILD inherits:

  • security identities
  • previous and current file descriptos
  • port and resource priveleges
  • envrionment variables
  • program code

    NOW THE CHILD CAN EXECUTE ITS OWN PROGRAM CODE!

2. NORMALLY PARENT SLEEPS WHILE CHILD RUNS

    wait request is set upon fork and signalled upon CHILD COMPLETION

3. When CHILD EXITS

  1. it closes or discards its resources and environment
  2. leaves a ZOMBIE RESOURCE (:= entry in process table)
  3. PARENT is SIGNALED TO WAKE & CLEANS process table of child entry.
  4. PARENT continues its own program code execution

** A process which has NO PARENT PROCESS is referred to as a zombie process

   

Describing Process States:

Each CPU/CPU-Core can run a single process at a time. Immediate process requirments for CPU time & resource allocation constantly change. As such the states of a process are also always changing as cirumstances dictate.

   

Linux Process States in Detail:

ps [OPTION] := report a snapshot of the current processes

    -aux := list EVERY process using BSD syntax
    -lax := provides a long listing and displays faster
    -ef  := display all processes
    -j   := displays information about jobs 
            ( PPID | PID | PGID | SID | TTY | TGPID | STAT | UID | TIME | COMMAND )
                                                       ^^(Process State Letter)

    -O [FORMAT] := sorts by given format

pstree [OPTION] := display a TREE of processes

    -g := show PGIDs
    -p := show PIDs

Control Jobs:

Any command OR pipeline can be STARTED IN BACKGROUND by appending an ampersand (&) to it. Bash displays job number (unique to session) and child process PID

jobs := dissplays the list of jobs for current shell session

fg %[JOB #]:= brings JOB $ into the foreground

bg %[JOB #]:= STARTS RUNNING SUSPENDED PROCESS JOB #$

 

** CTR-Z -> Generates a SUSPEND REQUEST for current fg process

Kill Processes:

    A signal is a software interrupt delivered to a PROCESS. Evens that can generate signals include an error, an external event (IO req. or expired timer) or by EXPLICTELY SENDING A SIGNAL.

Each signal has a DEFAULT BEHAVIOR, usually:

                1. Term: Terminate a program (exit) at once.

                2. Core: Save memory image (core dump), then terminate

                3. Stop: Stop a running program (suspend) and wait to continue (resume).

 

kill [OPTION] [NAME/PID of P to Kill] := terminate a process

    -s, --signal [signal] := provides the signal to send (given as name or number)


    -p, --pid := only print the PID of named process, DO NOT SEND SIGNAL
    --verbose := print PID(s) that will be signaled with kill and SEND SIGNAL

    ** Each PID can express as:

        n  := (n>0) | PID n is signalled

        0  := all processes in current process group are signalled

        -1 := all process with PID > 1 are signalled

        +n := (n>0) | All process in process group n are signalled

        NAME := process NAME will be signalled

 

killall [OPTION] [NAME] := kill processes by NAME

    -I, --ignore-case := ignore case when searching 
    -e, --exact := requires an EXACT match.
                   ** If a command is > 15 chars the full name may be unavailable at which
                      case it will KILLALL THAT MATCHES FIRST 15 CHARS. '-e' will SKIP
                      these entries but will specify which are skipped if using '-v'.

    -i, --interactive := ask for confirmation for each kill

 
 

pkill [OPTION] [PATTERN] := send signal to each process instead of listing to stdout

 

pgrep [OPTION] [PATTERN] := looks through currently running processes and lists the PIDs which match [PATTERM] to stdout

 

pidwait [OPTION] [PATTERN] := will WAIT for each process instead of listing to stdout

    -u, --euid [euid] := only match process whose EUID is given (numerical or sumbolic value)
    -U, --uid [uid]   := only match process whose UID is given (numerical or symbolic value)

    -g, --pgroup [pgroup] := only match process in PGIDs listed
    -G, --group [GID] := only match process in GIDs listed
    
pkill EXCLUSIVE:
    -e, --echo := disply NAME and PID of process being killed 
    -[SIGNAL] OR --signal [SIGNAL] := dictates which signal to send to each process

    -s, --short := display in short format
    -i, --ip-addr := display IP address instead of host name for 'from' field

2: Identify PID Numbers to Kill :

    Using pgrep we get all of 'bob's processes

    [root@host ~]# pgrep -l -u bob

    6998 sleep
    6999 sleep
    7000 sleep
    . . . . . .

3: Execute Kill Command:

    . . . . . .
    [root@host ~]# pkill -SIGKILL -u bob
    [root@host ~]# pgrep -l -u bob

Monitor Process Activity:

Load Average: The measurement Linux provides to represent percieved system load of a given process.

 
    Calculating Load Average:

        AVG(# of process in R (ready to run) state or D (waiting for I/O to complete) state.

        AVG(#R_processes + #D_processes)

 

uptime [OPTION] := Tell how long sys. has been running INCLUDING load average

    [user@host ~]$ uptime
    15:29:03 up 14 min,  2 users,  load average: 2.92, 4.48, 5.20top
                                  time(min) = .   ^5    ^10   ^ 15
