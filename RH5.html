I/O Streams (Standard Input, Output & Error)

 

When it comes to IO there exist THREE main channels:  
       0 := Standard Input, reads from keyboard  
       1 := Standard Output, sends to terminal  
       2 := Standard Error, sends errors to terminal  
Custom channels can also be created:  
       3+ := Custom channels with custom function  
 

Number Channel name Description Default connection Usage
0 stdin Standard input Keyboard read only
1 stdout Standard output Terminal write only
2 stderr Standard error Terminal write only
3+ filename Other files none read, write, or both

How To Redirect Output:

[COMMAND]   [ REDIRECTION]   [DESTINATION_FILE]

        [COMMAND]          := Some command with an output

        [REDIRECTION]      := Redirection character

        [DESTINATION_FILE] := Destination file path  
 
 

     cmd > path   : REDIRECTS & OVERWRITES stdout into file at path

     cmd >> path  : REDIRECTS & APPENDS stdout into file at path

     cmd 2> path  : REDIRECTS & OVERWRITES stderr into file at path

     cmd 2>/dev/null path : DISCARDS stderr by redirecting to '/dev/nul'

 

     {cmd > path 2>&1 : REDIRECTS & OVERWRITES stdout & stderr } OR

     {cmd &> path     : into the SAME file                     }

 

     {cmd >> path 2>&1 : REDIRECTS & APPENDS stdout & stderr } OR

     {cmd &>> path     : into the SAME file                  }

 
 

** The ORDER in which you perform REDIRECTION is also VERY IMPORTANT:

    Redirects stdout and THEN stderr to output.log
        > output.log 2>&1

    Redirects stderr to default place for stdout AND THEN REDIRECTS ONLY stdout to output.log
         2>&1 > output.log

Pipelines:

A pipeline is a sequence of one or more commands that are seperated by '|'.

Connects OUTPUT of first command to INPUT of second command.

Combining Pipelines & Redirection:

    When you combine, the shell sets up the ENTIRE PIPELINE FIRST and then REDIRECTS. Because of this, if you REDIRECT to FILE in the MIDDLE OF A PIPELINE, then <output goes to FILE NOT NEXT COMMAND

 

To circumvent this we use:

tee [OPTIONS] [FILE_PATH] := read from stdin and write to stdout AND FILE_PATH

cat [OPTION] [FILE/PATH] '[FILE/PATH]' -> Concatenate files and print to stdout
	-a  := append to the given FILES, do not overwrite
	-i  := ignore interrupt signals

Edit Text Files from Shell Prompt:

There are a number of different built-in ways of editing text files on linux systems. Some of those systems include nano and visudo. The most used and improved editor however is: vim

 

The biggest reason to use vim is that is is almost always installed by default on server for editing text-based files.

   
 

vi [FILE_PATH]  := opens [FILE_PATH] wih the basic vi editor.

vim [FILE_PATH] := opens [FILE_PATH] wih the advanced vim editor.

 

Vim Operating Modes:

        i := insert mode

        v := visual edit mode

        : := extended command mode

   ** If you are unsure which mode you are in, press ESC a few times to safely return to command mode.

MUST KNOW VIM COMMANDS:

        u key := UNDOES most recent edit.

        x key := DELETES SINGLE CHARACTER

 

        :w  := writes (saves) the file and remains in command mode for editing.

        :wq := writes (saves) the file and quits VIM.

        :q! := quits VIM and DISCARDS ALL CHANGES.

 

REARRANGE & SELECT EXISTING TEXT:

    Character Mode: v

    Line Mode     : Shift+v

    Block Mode    : Ctrl+v

Setting & Editing Environment Variables:

Exporting a Shell Var into an Environmental Var:

    [...]$ export EDITOR
              OR
    [...]$ export EDITOR=vim

    Examples of Environmental Variables:

    ~ Changing Language of Environment:
        [user@host ~]$ date

        [user@host ~]$ export LANG=fr_FR.UTF-8
        [user@host ~]$ date
        mar. janv. 22 16:38:14 CST 2019

    ~ Changing Where Commands are Found in Shell
        $PATH := a list of colon-seperated dirs that contain programs
        ** When you run a command in a terminal it looks at these directories for what to do.

        To Append to $PATH:
            [user@host ~]$ export PATH=${PATH}:/home/user/sbin

    ~ Set Default Text Editor:
        [user@host ~]$ export EDITOR=nano

Setting Variables Automatically:

When BASH starts, several text files run with shell commands that initialize the environment. To set shell/environmental vars automatically on shell start up, you can EDIT BASH STARTUP SCRIPTS  
 
What determines exactly which startup script is used are 2 things:

        1. INTERACTIVE OR NON-INTERACTIVE SHELL

        2. LOG-IN OR NON-LOGIN SHELL    

For interactive log-in shell     : /etc/profile AND ~/.bash_profile

                                 : (/etc/bashrc AND ~/.bashrc also sourced respectively)

For interactive non-log-in shell : ONLY /etc/bashrc AND ~/.bashrc  
 

**/etc/profile AND ~/.bash_profile are USER-SPECIFIC, while /etc/bashrc AND ~/.bashrc APPLY TO WHOLE SYSTEM
